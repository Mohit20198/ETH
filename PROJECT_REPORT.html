Section 01

Problem Statement

Industrial plants and manufacturing facilities generate an enormous volume of unstructured and siloed data — P&IDs (Piping & Instrumentation Diagrams), maintenance logs, safety inspection reports, compliance manuals (OSHA, OISD), and incident records. This data lives in disconnected PDFs, spreadsheets, and legacy databases.

⏱️
Prolonged Downtime

Engineers waste hours searching disconnected PDFs during Root Cause Analysis (RCA), extending equipment downtime.

⚠️
Compliance Risk

Manual verification of safety regulations (OSHA, OISD, Factory Act) is error-prone and increases audit risk.

🔁
Recurring Failures

Lessons learned from past incidents are buried in old reports, leading to the same failures repeating.

🧠
Knowledge Loss

Institutional knowledge leaves with experienced operators. New engineers have no structured way to access it.

Section 02

Our Solution: IndustrialIQ

IndustrialIQ is a multi-modal, multi-agent AI platform that ingests industrial documents of any format, extracts structured knowledge into a graph database, and enables plant engineers to query it in natural language — receiving accurate, cited, and confidence-scored answers in seconds.

The system unifies semantic vector search (ChromaDB) with graph-based relationship traversal (Neo4j) through a weighted hybrid retrieval engine, then routes queries to specialized AI agents orchestrated by a LangGraph supervisor.

📄
Multi-Modal Ingestion

Processes PDF, DOCX, XLSX, TXT, PNG/JPG (P&IDs), and EML files in a unified pipeline.

👁️
Vision & OCR

GPT-4 Vision + Tesseract OCR extracts text and relationships from engineering diagrams.

🕸️
Knowledge Graph

Entities and relationships extracted and stored in Neo4j with full provenance and confidence scores.

🔍
Hybrid Retrieval

ChromaDB vector search combined with Neo4j graph traversal and a weighted reranker.

🤖
Multi-Agent AI

LangGraph supervisor dispatches to Maintenance, Compliance, Copilot, and Lessons Learned agents in parallel.

✅
Traceability

Every answer is backed by source citations with document names and confidence percentages.

Section 03

System Architecture

Query Pipeline

User Query
    │
    ▼
FastAPI Gateway (backend/api/main.py)
    │
    ▼
LangGraph Supervisor (backend/agents/supervisor.py)
    │
    ├─ 1. classify_node ─── Query Classifier (single-fact / multi-hop / aggregation)
    │
    ├─ 2. retrieve_node ─┬── ChromaDB Vector Search (always runs)
    │                   └── Neo4j Graph Traversal (multi-hop / low-confidence fallback)
    │                         └─ Weighted Reranker (vector=0.6, graph=0.4)
    │
    ├─ 3. dispatch_node ─── Specialist Agents (parallel, immutable context)
    │                  ├── CopilotAgent          (general Q&A)
    │                  ├── MaintenanceAgent   (RCA, MTBF, work orders)
    │                  ├── ComplianceAgent    (OSHA, OISD, Factory Act)
    │                  └── LessonsLearnedAgent(failure patterns, history)
    │
    └─ 4. synthesize_node ── Final answer + citations + confidence score
            │
            ▼
AgentResponse { answer, citations, confidence, graph_path, query_type, latency_ms }

Document Ingestion Pipeline

New Document
    │
    ├─ Fingerprint Check (SHA-256) ── already seen? → SKIP
    │
    ├─ Parse (per-type router)
    │   ├── PDF       → pdfplumber + OCR fallback
    │   ├── P&ID/IMG → OpenCV preprocess + pytesseract + GPT-4V
    │   ├── Excel    → openpyxl (spatial layout preserved)
    │   ├── DOCX     → python-docx
    │   └── TXT/EML → direct read
    │
    ├─ Semantic Chunking
    ├─ Embed chunks → ChromaDB
    │
    └─ Two-Pass Extraction → Neo4j
        ├── Pass 1: LLM proposes {nodes, edges} against pinned ontology
        └── Pass 2: Verifier assigns confidence, rejects < 0.70

Section 05

Knowledge Graph Ontology

The shared ontology (backend/shared/ontology.py) is a locked contract that all team members depend on and cannot change without team agreement.

Node TypesEdge Types
Equipment, Document, Parameter, Person, Regulation, Event HAS_PARAMETER, DOCUMENTED_IN, PERFORMED_BY, PRECEDED_BY, REGULATED_BY, CONNECTED_TO, MENTIONS, CAUSED_BY

Every edge carries full provenance: source document ID, confidence score, extraction pass ID, timestamp, and verified flag. Only edges with confidence ≥ 0.70 are written to the graph.
